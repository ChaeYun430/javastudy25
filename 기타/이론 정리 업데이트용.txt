다형성(多形性)
부모를 통해 다변하는 성질
같은 타입 이지만 실행 결과가 다양한 객체를 이용할 수 있는 성질
코드 측면에서 보면 다형성은 하나의 타입에 여러 객체를 대입함으로써 다양한 기능을 이용할 수 있도록 해준다.
//부모 타입의 멤버를 사용하지만 자식 클래스에서 오버라이딩된 멤버는 대체하여 사용
//부모 타입의 멤버(자식 클래스에서 오버라이딩된 멤버 제외) + 자식 클래스에서 오버라이딩된 멤버

다형성을 위해서 자바는 부모 클래스로 타입 변환을 허용한다. (강제타입변환, 자동타입변환)
부모 타입에 모든 자식 객체가 대입될 수 있다.
객체는 부품화가 가능


자식 타입이 부모 타입으로 자동 변환 후, 다시 자식 타입으로 변환 할 때 강제 타입 변환을 사용(자동타입변환 -> 강제타입변환-> 부모사용)
자식 클래스 변수 = (자식클래스) 부모클래스타입;
부모 클래스타입 : 자식 타입이 부모타입으로 변환된 상태


타입 변환 : 데이터 타입을 다른 데이터 타입으로 변환하는 행위
  - 상속 관계에 있는 클래스 사이에서 발생


자동 타입 변환
자식 타입은 부모 타입으로 자동 타입 변환이 가능
자식은 부모의 특징과 기능을 상속 받기 때문에 부모와 동일하게 취급될 수 있다는 것
바로위의 부모가 아니더라도 상속 계층에서 상위 타입이라면 자동 타입 변환이 일어날 수 있다.

부모 타입으로 자동타입 변환된 이후에는 부모 클래스에 선언된 필드와 메소드만 접근이 가능하다. (method3이 안됨)
변수는 자식 객체를 참조하지만 변수로 접근 가능한 맴버는 부모 클래스 맴버로 한정된다.
예외는 메소드가 자식 클래스에서 오버라이딩 되었다면 자식 클래스의 메소드가 대신 호출된다.(다형성과 관련됨)
